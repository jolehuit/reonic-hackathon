## 🤔 What is Pioneer?

Pioneer is an agentic fine-tuning and inference platform for small and large open source models that allows you to do the following through our conversation agent or API:

- Generate high quality synthetic training data
- Fine-tune open source SLMs and LLMs for specific tasks and domains
- Run custom evals against base models, GPT-4o and industry standard benchmarks
- Deploy fine-tuned models on Pioneer
- Download your own model weights

## 🧰 Why build with Pioneer?

With Pioneer, you can fine-tune SLMs capable of matching or beating frontier LLMs on specific tasks and walk away with your own model weights. This has several implications for your build:

- High quality synthetic data generation and fine-tuning allows for more precise results compared to prompt engineering
- Low-latency SLMs are optimized for agentic workflows
- Download and own all your fine-tuned models
- Fine-tuned encoder models that provide reliable, structured outputs with confidence scores
- SLMs can be run locally with ease 🦞

### 🤏 GLiNER2

[GLiNER2](https://arxiv.org/pdf/2507.18546) is a 205M parameter encoder-based micro language model. It has several unique qualities that, when fine-tuned on specific tasks, make it a competitive replacement for expensive frontier LLMs.

- **Deterministic outputs.** Because GLiNER2 isn’t a generative model, its outputs are structured and consistent (no hallucinations, no malformed JSON, no parsing failures). It also gives confidence scores for extracted entities and classifications.
- **Fast and effective fine-tuning.** Fine-tuning on even 100 additional examples can improve performance dramatically. And because the model is so small, fine-tuning on 5k examples (5 epochs) can take as little as 20 minutes.
- **Handles multiple tasks in a single forward pass.** Means that adding more tasks and more labels to the model doesn’t add any latency.
- **Can be run locally.** GLiNER2 models are only ~205 million parameters and inference can run on CPU only. You can also download your weights and run it on your own infrastructure, which makes it ideal for processing highly sensitive and confidential data.

We recommend fine-tuning our **GLiNER2** encoder-based model (205M parameters) for near frontier-level performance (at a fraction of the size, cost, and latency) for tasks such as:

- Text classification
- JSON extraction
- Named Entity Recognition
- Relation Extraction

Voici une description détaillée du processus étape par étape présenté dans la vidéo :

1. Requête initiale et génération des données

L'utilisateur interagit avec l'agent IA sur la plateforme "Pioneer" et lui demande de créer (fine-tuner) un modèle capable de classer les requêtes des utilisateurs selon deux catégories : celles nécessitant un modèle de raisonnement complexe et celles pouvant être traitées par un modèle plus petit (non-raisonnement).

L'agent commence par générer un jeu de données synthétique de 400 exemples pour l'entraînement, divisés entre les étiquettes reasoning et non-reasoning.

2. Nettoyage et préparation des données

L'agent exécute de manière autonome un script Python pour inspecter la qualité des données générées.

Il identifie 15 exemples "bruyants" ou problématiques (ex: des concepts abstraits au lieu de vraies questions) et les supprime.

Le jeu de données final, contenant 385 exemples propres, est téléchargé pour l'entraînement.

3. Entraînement (Fine-tuning) du modèle

L'agent lance une tâche d'entraînement en utilisant le modèle de base fastino/gliner2-base-v1.

L'interface affiche les paramètres d'entraînement (5 époques, taux d'apprentissage de 3e-5, etc.). Le statut du travail passe de "demandé" à "déployé".

4. Évaluation du modèle

Une fois l'entraînement terminé, l'agent exécute un script d'évaluation sur un échantillon de validation (77 requêtes).

Il effectue un balayage des seuils de confiance et analyse les erreurs (faux positifs et faux négatifs).

Le modèle atteint une précision de 92,2%, soit une amélioration de 22 points par rapport au modèle de base.

5. Déploiement sur l'API

L'utilisateur demande à l'agent de déployer le modèle sur l'API d'inférence de Pioneer.

L'agent exécute les commandes nécessaires et confirme le déploiement du modèle nommé query-router-v1. Il fournit également un exemple de commande curl montrant comment interroger le modèle via l'API.

6. Test et comparaison dans le Playground

L'utilisateur navigue vers l'onglet Playground dans le menu de gauche pour tester le modèle en direct.

Il sélectionne 4 modèles différents pour les comparer côte à côte (GLiNER2 Base V1, Llama 3.3 70B, Qwen3 8B, et le modèle fraîchement entraîné query-router-v1).

Il configure le schéma d'inférence en ne gardant que la classification reasoning / non-reasoning.

L'utilisateur tape la requête test "How many r's in strawberry?" (Combien de 'r' dans fraise ?) et lance l'exécution.

Les résultats s'affichent pour chaque modèle. Le modèle personnalisé (query-router-v1) classe avec succès la requête dans la catégorie "Non-reasoning" avec 100% de confiance, démontrant que le fine-tuning a fonctionné. Un tableau en bas d'écran compare également la latence des différents modèles.

def make_text(p):
    has_ev = p['has_ev'] == 'yes'
    r = random.random()
    if r < 0.60:
        templates = DE_TEMPLATES_EV if has_ev else DE_TEMPLATES_NO_EV
    elif r < 0.90:
        templates = EN_TEMPLATES_EV if has_ev else EN_TEMPLATES_NO_EV

def find_span(text, value_str):
    """Find exact substring in text."""
    idx = text.find(value_str)
    if idx == -1:
        return None
    return value_str

def build_entities(text, p):
    entities = []
    
    # inhabitants_count
    v = str(p['inhabitants_count'])
    if find_span(text, v):
        entities.append([v, 'inhabitants_count'])
    
    # house_size_sqm
    v = str(p['house_size_sqm'])
    if find_span(text, v):
        entities.append([v, 'house_size_sqm'])
    
    # annual_consumption_kwh
    v = str(p['annual_consumption_kwh'])
    if find_span(text, v):
        entities.append([v, 'annual_consumption_kwh'])
    
    # ev_annual_km (only if EV)
    if p['has_ev'] == 'yes' and p.get('ev_annual_km') is not None:
        v = str(p['ev_annual_km'])
        if find_span(text, v):
            entities.append([v, 'ev_annual_km'])
    
    # has_ev — find the yes/no signal word in text
    if p['has_ev'] == 'yes':
        for kw in ['E-Auto', 'Elektroauto', 'E-Fahrzeug', 'electric vehicle', 'EV', 'VE', 'véhicule électrique']:
            if find_span(text, kw):
                entities.append([kw, 'has_ev'])
                break
    else:
        for kw in ['kein Elektroauto', 'no EV', 'sans véhicule électrique']:
            if find_span(text, kw):
                entities.append([kw, 'has_ev'])
                break
    
    # heating_type — find the heating word
    heating_map = {
        'oil': ['Öl', 'Ölheizung', 'oil', 'fioul'],
        'gas': ['Gas', 'Gasheizung', 'gas', 'gaz'],
        'heatpump': ['Wärmepumpe', 'Wärmepumpenheizung', 'heat pump', 'heatpump', 'pompe à chaleur'],
        'other': ['Sonstige', 'Sonstigeheizung', 'other', 'autre'],
    }
    for kw in heating_map.get(p['heating_type'], []):
        if find_span(text, kw):
            entities.append([kw, 'heating_type'])
            break
    
    return entities
